## 1. 전체 아키텍처

```
[개발자 PC]
    │ git push origin main
    ▼
[GitHub Repository]
    │ GitHub Actions 트리거
    ▼
[Linux Server]  ← SSH 접속 → deploy.sh 실행
  ├── Nginx (80/443)          [diary-external 네트워크]
  ├── Next.js App (:4000)     [diary-internal 네트워크]
  └── MariaDB (:3306)         [diary-internal 네트워크]
```

**배포 흐름**: `git push main` → GitHub Actions → SSH → `git pull` → `docker compose build` → `docker compose up -d`

## 2. 사전 준비

### 2-1. 로컬 코드를 GitHub에 올리기

현재 코드가 로컬에만 있으므로 먼저 GitHub에 올려야 서버에서 클론할 수 있습니다.

```powershell
# 1. GitHub에서 새 Repository 생성 (github.com → New repository)
#    - Repository name: family-diary (또는 원하는 이름)
#    - Visibility: Private (가족 전용 서비스이므로 비공개 권장)
#    - Initialize 옵션 모두 체크 해제 (코드가 이미 있으므로)

# 2. 로컬 프로젝트에 원격 저장소 연결
cd C:\Develop\다이어리

# 원격 저장소 추가 (YOUR_USERNAME, YOUR_REPO 를 실제 값으로 교체)
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# 3. main 브랜치로 전환 (현재 master인 경우)
git branch -M main

# 4. 최초 push
git push -u origin main
```

> ⚠️ `.env.local` 파일은 `.gitignore`에 포함되어 있어 push 되지 않습니다. (민감 정보 보호)

### 2-2. 로컬 환경 (Windows)

- [ ] 위 단계에서 GitHub Repository 생성 및 push 완료
- [ ] 로컬 MariaDB에서 DB 덤프 파일 준비
- [ ] SSH 키 페어 생성

```powershell
# SSH 키 생성 (PowerShell)
ssh-keygen -t ed25519 -C "github-actions-deploy" -f "$HOME\.ssh\deploy_key"
# 생성된 파일:
#   deploy_key      (비공개키 → GitHub Secret에 등록)
#   deploy_key.pub  (공개키  → 서버에 등록)
```

### 2-2. 리눅스 서버 요구사항

| 항목 | 최소 사양 |
|------|-----------|
| OS | Ubuntu 22.04 LTS 이상 |
| CPU | 2 Core |
| RAM | 2 GB |
| Disk | 20 GB |
| 포트 개방 | 22 (SSH), 80 (HTTP), 443 (HTTPS) |

## 3. 서버 초기 설정

### 3-1. Docker 설치

```bash
# Ubuntu 22.04 기준
sudo apt update && sudo apt upgrade -y

# Docker 공식 설치 스크립트
curl -fsSL https://get.docker.com | sudo sh

# 현재 사용자를 docker 그룹에 추가 (sudo 없이 docker 명령 사용)
sudo usermod -aG docker $USER
newgrp docker

# 설치 확인
docker --version
docker compose version
```

### 3-2. Git 설치

```bash
sudo apt install -y git
git --version
```

### 3-3. GitHub Actions용 SSH 공개키 등록

```bash
# 서버에서 실행
mkdir -p ~/.ssh
chmod 700 ~/.ssh

# 로컬에서 생성한 deploy_key.pub 내용을 붙여넣기
echo "ssh-ed25519 AAAA...공개키내용..." >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

### 3-4. 프로젝트 디렉토리 생성 및 클론

> 2-1 단계에서 GitHub에 push가 완료된 후 진행합니다.

```bash
sudo chown $USER:$USER /opt/family-diary

# GitHub에서 클론 (YOUR_USERNAME, YOUR_REPO 를 실제 값으로 교체)
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git /opt/family-diary

cd /opt/family-diary
```

## 4. GitHub Repository 설정

### 4-1. GitHub Secrets 등록

GitHub Repository → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

| Secret 이름 | 값 |
|-------------|-----|
| `SSH_HOST` | 서버 IP 또는 도메인 (예: `192.168.1.100`) |
| `SSH_USER` | 서버 SSH 사용자명 (예: `ubuntu`) |
| `SSH_PRIVATE_KEY` | `deploy_key` 파일 전체 내용 (-----BEGIN부터 끝까지) |
| `SSH_PORT` | SSH 포트 (기본: `22`) |

### 4-2. main 브랜치 보호 설정 (선택)

GitHub Repository → **Settings** → **Branches** → **Add rule**
- Branch name pattern: `main`
- Require pull request before merging: ✅ (선택사항)

## 5. GitHub Actions CI/CD 구성

`.github/workflows/deploy.yml` 파일이 이미 프로젝트에 포함되어 있습니다.

### 배포 트리거

```
git push origin main  →  자동 배포 시작
```

### 배포 흐름 (자동)

```
1. GitHub Actions 워크플로 실행 (ubuntu-latest runner)
2. appleboy/ssh-action으로 서버 SSH 접속
3. /opt/family-diary/deploy/linux/scripts/deploy.sh 실행
   ├── DB 백업
   ├── git pull origin main
   ├── docker compose build next-app
   ├── docker compose up -d next-app
   └── 헬스체크 확인
```

### 배포 상태 확인

GitHub Repository → **Actions** 탭에서 실시간 로그 확인 가능

## 6. 데이터베이스 마이그레이션

> 로컬 Windows의 MariaDB 데이터를 리눅스 서버로 이전

### 6-1. 로컬 DB 덤프 (Windows PowerShell)

```powershell
# MariaDB 설치 경로에 맞게 조정
$mysqldump = "C:\Program Files\MariaDB 11.8\bin\mysqldump.exe"
$dumpFile  = "C:\Users\$env:USERNAME\Desktop\family_planner_dump.sql"

& $mysqldump -u root -p740923aa `
    --single-transaction `
    --routines `
    --triggers `
    --databases family_planner > $dumpFile

Write-Host "덤프 완료: $dumpFile"
```

### 6-2. 덤프 파일 서버 전송

```powershell
# PowerShell에서 scp로 전송
scp -i "$HOME\.ssh\deploy_key" `
    "C:\Users\$env:USERNAME\Desktop\family_planner_dump.sql" `
    ubuntu@YOUR_SERVER_IP:/opt/family-diary/family_planner_dump.sql
```

### 6-3. 서버에서 MariaDB 컨테이너 기동 (앱 제외)

```bash
cd /opt/family-diary/deploy/linux

# MariaDB만 먼저 기동
docker compose up -d mariadb

# DB 준비 완료까지 대기 (약 30초)
sleep 30
docker compose ps mariadb   # healthy 상태 확인
```

### 6-4. 덤프 파일 임포트

```bash
# 덤프 파일을 컨테이너로 복사
docker cp /opt/family-diary/family_planner_dump.sql family-diary-db:/tmp/dump.sql

# 컨테이너 안에서 임포트 실행
docker exec -i family-diary-db sh -c \
  "mariadb -u root -p\${MARIADB_ROOT_PASSWORD} < /tmp/dump.sql"

# 임포트 확인
docker exec family-diary-db mariadb -u root -p${DB_ROOT_PASSWORD} \
  -e "SELECT COUNT(*) AS user_count FROM family_planner.users;"
```

> ⚠️ `${DB_ROOT_PASSWORD}` 는 `deploy/linux/.env` 에 설정한 값과 동일해야 합니다.

### 6-5. 덤프 파일 정리

```bash
rm /opt/family-diary/family_planner_dump.sql
docker exec family-diary-db rm /tmp/dump.sql
```

## 8. 도메인 및 SSL 설정

### 방법 A: Cloudflare Tunnel (권장 — 포트 개방 불필요)

```bash
# Cloudflare Tunnel 설치
curl -L --output cloudflared.deb https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared.deb

# Cloudflare 로그인 및 터널 생성
cloudflared tunnel login
cloudflared tunnel create family-diary
cloudflared tunnel route dns family-diary your-domain.com

# 서비스 등록 (자동 시작)
cloudflared service install
```

`deploy/linux/nginx/conf.d/app.conf` 에서 **MODE B (HTTP only)** 활성화:
```nginx
# Mode A 전체 주석 처리
# Mode B 주석 해제
```

### 방법 B: Let's Encrypt (직접 포트 개방)

```bash
# Certbot 설치
sudo apt install -y certbot

# 인증서 발급 (nginx 중지 후)
docker compose stop nginx
sudo certbot certonly --standalone -d your-domain.com

# 인증서 복사
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem \
    /opt/family-diary/deploy/linux/nginx/certs/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem \
    /opt/family-diary/deploy/linux/nginx/certs/

docker compose start nginx
```

## 11. 트러블슈팅

### 배포 실패 — GitHub Actions SSH 연결 오류

```
Error: dial tcp: connection refused
```

→ 서버 방화벽에서 SSH 포트(22) 허용 확인
```bash
sudo ufw allow 22
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
```

### Next.js 빌드 실패 — 메모리 부족

```bash
# swap 파일 추가 (2GB)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### DB 연결 오류 — depends_on 타이밍 문제

```bash
# MariaDB healthcheck 통과 전 앱 기동 시 발생
# 해결: 앱 컨테이너만 재시작
docker compose restart next-app
```

### 텔레그램 알림 미발송

```bash
# cron 등록 확인
docker logs family-diary-app | grep "Telegram"
# "[Telegram] 일일 브리핑 스케줄러 등록됨" 로그 확인

# 수동 발송 테스트
curl -X POST http://localhost:4000/api/cron/daily-brief \
  -H "Authorization: Bearer ${CRON_SECRET}"
```
